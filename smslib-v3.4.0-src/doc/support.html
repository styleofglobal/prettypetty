Support

SMSLib News

The SMSLib Blog is used for announcing new releases, bugs / fixes and other important stuff about SMSLib. If you work with SMSLib, you are advised to keep in touch with the blog.

SMSLib User Group

The SMSLib User Group is the place to request support, provide feedback and discuss about SMSLib with the developers and other users.

The following simple rules govern your participation in the SMSLib User Group:

  • Be polite.
  • Do not post the same question more than once.
  • Do not hijack discussions. If you have a new issue, start a new thread. Do not post or answer to irrelevant threads.

Posts not following the above rules will be deleted without warning. Posters who overlook the rules will be banned. Lets try to keep the group useful and consice!

Please note that messages from new members are moderated (spam screening). This is not done to change the content of message - this is to ensure that spam messages do not reach the group. The moderation status is removed after the first couple of messages are reviewed by hand. I am sorry for the inconvinience, but I hate spam... and the list gets plenty of it!

SMSLib Project Updates

The SMSLib Project Updates Group receives automatic notification emails about the activity of the repository, the tracker and wiki comments. Posting is not allowed on this group. This is not a public group (visitors cannot view posts), however everyone is free to join.

Issue Tracker

You can use the SMSLib Issue Tracker to file bugs, submit patches or feature requests (note: Google account required).

Repository

SMSLib Subversion repository is currently hosted on Google Code.

You can checkout the current SMSLib version from the following path: http://smslib.googlecode.com/svn/smslib/trunk/

Please note that the repository snapshots may not always work properly. We try our best to commit codes that works, but don't forget that the repo is for development... If you are interested in stable versions, you are advised to following the published releases.
