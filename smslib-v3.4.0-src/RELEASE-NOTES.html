SMSLib v3.4.0 (Mar 29th, 2009)
Release Notes


Upgrade path from v3.3.3

Please note the following issues while planning your upgrade:

  • SMSLib v3.4 is incompatible with the previous v3.3.3. The changes are not much though, and you will be able to fix them by just reviewing the compile errors.
  • The logging subsystem has changed from log4j to slf4j. Please review the Installation Instructions before proceeding.
  • New fields have been added in the database tables (SMSServer Database Interface). Please review the relevant DB Interface page and adjust your database accordingly.
  • Sample DB scripts for Microsoft SQL Server, MySQL and Oracle are now available in the /misc/SMSServer/Database Scripts/ directory.
  • The JMX, Email and XML SMSServer interfaces have been deprecated. You are advised to switch to the Database Interface if possible.
  • In an attempt to reduce the dependencies for compiling SMSLib/SMSServer, some SMSServer interfaces have been moved out of the main source tree. These interfaces can be found in the /misc/SMSServer/Interfaces/ directory. Please follow the installation instructions found on each interface documentation page if you want to install and work with them.

Some important bugs found in v3.3.3 have been correct in this release - you are advised to upgrade the soonest possible.

  6. SMSLib: New ATHandler for Wavecom SunTraveller CDMA modem (contributed by Ernas Moethar) (#144)

  9. SMSLib: LeastLoadLoadBalancer is no longer available.
  10. SMSLib: Added basic phonebook management (see this page)
  11. SMSLib: Test Gateway did not setup a Gateway ID (#163)
  12. SMSLib: Fixed condition which could lead to NullPointer & NumberFormat exceptions for some non-compliant phones (#167)
  13. SMSLib: Implemented the Group Sending functionality (#169, see this page)
  14. SMSLib: New MC35i modem AT Handler (#179)
  15. SMSLib: Binary telnet option for IP modems now available (#180)
  16. SMSServer: Error in db interface when auto-delivery operation is enabled (#183)
  17. SMSServer: DB errors go unnoticed (#184)
  18. SMSLib: TPDU Data Coding Group Issue (#187)
  19. SMSLib: Flash messaging did not work as expected (#188)
  20. SMSServer: Add wap si capability via the SMSServer database interface (#185)
  21. SMSServer: Added "smsc_number" in configuration file / gateway definition for serial and IP modems (#193)
  22. SMSServer: Email interface issues(#191)
  23. SMSServer: If no directives are given, the configuration file is looked upon in the current directory
  24. SMSServer: JVM hanged when attempting to stop SMSServer with Ctrl-C (#195)
  25. SMSLib: Do not try to register to GSM network for ever - throw GatewayException after trying for a couple of times (#194)
  26. SMSServer: Set explicit fields in SQL-SELECT statements (#190)
  27. SMSServer: When no directives are given, SMSServer searches for the configuration file in the current directory.
  28. SMSLib: Renamed and added more Gateway Status fields (AGAteway.GatewayStatuses enumeration)

For more information, please visit the SMSLib Web Site.
